### 1. 📁 COMPLETE PROJECT STRUCTURE & CORE FILES

#### 🖥️ Frontend Files (under `/frontend`)

* **`frontend/index.html`**: The single HTML page shell that mounts the compiled React root application nodes.
* **`frontend/src/main.jsx`**: Bootstraps the application, registering the React virtual DOM renderer and importing global styles.
* **`frontend/src/App.jsx`**: The root React component managing user session state validation and global route transitions.

* **`frontend/src/pages/LandingPage.jsx`**: The public homepage showcasing app features with a prominent Neo-Brutalist "Get Started" entry button.
* **`frontend/src/pages/LoginPage.jsx`**: Handles credentials input, backend authentication requests, and JWT token storage.
* **`frontend/src/pages/SignupPage.jsx`**: Manages new account registration requests, salted hashing handshakes, and success redirects.
* **`frontend/src/pages/ProjectsPage.jsx`**: Lists, loads, saves, and deletes cloud projects from the Supabase database.
* **`frontend/src/utils/auth.js`**: Stores local login tokens, parses payload metadata, and handles server-side session verifications.
* **`frontend/src/utils/projects.js`**: Exports wrappers to call projects database API routes with bearer token headers.
* **`frontend/src/utils/timeAgo.js`**: Formats database timestamp objects into readable relative indicators (e.g., "3 minutes ago").
* **`frontend/src/components/layout/AppLayout.jsx`**: Sets up the dashboard workspace frame, combining sidebars, headers, and main scroll views.
* **`frontend/src/components/layout/Header.jsx`**: Shows the logo, active project name, save indicators, and user account actions.
* **`frontend/src/components/layout/Sidebar.jsx`**: Features tabs for navigating the data prep, summary, ML, and DL workspaces.

#### ⚙️ Backend Files (under `/backend`)
* **`backend/app.py`**: Boots the Flask server, configures CORS wildcards, and registers API blueprints.
* **`backend/db.py`**: Handles low-level HTTP rest requests to Supabase tables, bypasses RLS with service keys, and self-heals schemas.
* **`backend/middleware.py`**: Custom JWT authorization wrapper securing routes with token verification checks.
* **`backend/requirements.txt`**: Definitive production python dependencies manifest specifying required ML, DL, and server modules.
* **`backend/Dockerfile`**: Configures the non-root UID 1000 container sandbox environment, ports, and Gunicorn web server execution.
* **`backend/api/auth.py`**: Implements salted password storage via Bcrypt and issues signed authorization JWTs.
* **`backend/api/projects.py`**: Secure database routes to fetch, save, and delete projects from the user profiles.
* **`backend/api/upload.py`**: Parses uploaded CSV/Excel files into memory DataFrames.
* **`backend/api/clean.py`**: Route endpoint triggering data processor imputation, trimming, and winsorization.
* **`backend/api/metrics.py`**: Computes statistics (averages, standard deviations, categoricals) for data prep columns.
* **`backend/api/charts.py`**: Returns JSON plotting coordinates for data visualization tasks.
* **`backend/api/summary.py`**: Generates high-level statistical summaries, missingness heatmaps, and cell counts.
* **`backend/api/ml.py`**: Route controller linking ML configurations to the classical scikit-learn training engines.
* **`backend/api/dl.py`**: Endpoint managing sequential ANN compilation, fitting progress, and inference routing.
* **`backend/api/demo.py`**: Loads sample datasets (Titanic, Iris, etc.) into active memory for presentations.
* **`backend/api/predict.py`**: Runs real-time prediction evaluations using trained ML model files.
* **`backend/api/features.py`**: Handles feature engineering actions like scaling, encoding, and column binning.
* **`backend/api/dashboard.py`**: Calculates aggregation math and parses coordinates for visual dashboard widgets.
* **`backend/services/data_processor.py`**: Low-level engine carrying out pandas manipulations, scaling, and category maps.
* **`backend/services/ml_engine.py`**: Prepares training folds, runs Scikit-Learn estimators, and extracts feature weights.
* **`backend/services/dl_engine.py`**: Compiles TensorFlow models, feeds training datasets, and outputs real-time loss tracking logs.
* **`backend/services/inference.py`**: Performs real-time predictions by feeding raw arrays into compiled model objects.
* **`backend/services/model_store.py`**: In-memory registry keeping active trained models cached for inference calls.
* **`backend/services/visualizer.py`**: Renders correlation heatmaps and histograms, converting them to Base64 image files.

### 2. 📚 TECH STACK & LIBRARIES USED

#### 📦 Python Stack (`requirements.txt`)
* **`flask`**: Serves as the web framework to define backend routing, request mapping, and API endpoints.
* **`flask-cors`**: Handles cross-origin requests, allowing our Vercel-hosted frontend to bypass browser CORS blockages.
* **`supabase`**: Database client wrapper to write, select, and delete user project records in the Supabase PostgreSQL database.
* **`python-dotenv`**: Resolves local environment properties, injecting hidden service keys and secret strings into memory variables.
* **`bcrypt`**: Performs salted passwords hashing on account registration and compares credentials on login.
* **`pyjwt`**: Generates encrypted JSON Web Tokens (JWT) for secure session authentication.
* **`numpy`**: Provides optimized array objects and vectorized operations for fast data transformations.
* **`pandas`**: Key data manipulation library, loading CSVs/Excel files and handling data cleaning.
* **`scipy`**: Supplies mathematical modules for calculating skewness, kurtosis, and correlation matrices.
* **`matplotlib`**: Low-level rendering library to plot backend data charts.
* **`seaborn`**: High-level data visualization engine built on matplotlib for generating heatmaps.
* **`scikit-learn`**: Core machine learning package, supplying classical models (SVM, KNN, Decision Trees).
* **`tensorflow`**: High-performance deep learning ecosystem used to construct, fit, and predict sequential neural networks.

#### 🌐 Web Stack (`package.json`)
* **`react`**: Component-oriented interface library used to manage states, layouts, and page routing.
* **`react-dom`**: Mounting framework rendering React component trees into browser DOM nodes.
* **`react-router-dom`**: Frontend routing engine directing pages (landing, login, workspace) without page reloads.
* **`axios`**: Promise-based HTTP package used to send data payloads to backend routes.
* **`recharts`**: Dynamic React chart library generating scalable SVG dashboards.
* **`react-grid-layout`**: Grid controller facilitating drag-and-resize features for widgets.
* **`lucide-react`**: Vector icon library providing clean graphical designs.
* **`html2canvas`**: Converts dashboard container divisions into images for export.
* **`jspdf` & `jspdf-autotable`**: Generates high-resolution PDF document reports from project state objects.

### 3. ⚙️ PIPELINE FEATURE BREAKDOWN

#### A. Data Ingestion & Preprocessing (PREP Tab)
* **File Upload**: Parses file binaries using Pandas `read_csv()` and `read_excel()`, checking schema structures.
* **Missing Value Handling**: Replaces null entries using `fillna()` with Mean, Median, Mode, custom strings, or drops rows via `dropna()`.
* **String Cleaning**: Performs string cleaning via pandas `.str` accessor methods (trimming, case mapping, regex character removal).
* **Outlier Detection**: Detects values outside IQR bounds: `[Q1 - 1.5*IQR, Q3 + 1.5*IQR]`.
* **Outlier Handling**: Removes outlier rows or caps values within IQR boundaries.
* **Action Queue**: Stages data actions in memory, allowing users to verify changes before writing to the server.
* **Schema Management**: Converts column formats manually (e.g. mapping numeric columns or date variables).

#### B. Descriptive Analytics (SUMMARY Tab)
* **High-Level Statistics**: Calculates dataset shapes, duplicate rows, missingness ratios, and data sizes.
* **Descriptive Mappings**: Uses `df.describe()` to display metrics (mean, std, quartiles) for numeric fields, and class frequencies for text.
* **Missingness Heatmaps**: Plots null density matrices, encoding empty coordinates as dark blocks.

#### C. Variable Transformation (FEATURES Tab)
* **Formula Creation**: Generates new fields using math operations (+, -, *, /) or conditional checks.
* **Feature Deletion/Renaming**: Removes redundant fields using `df.drop()`, or renames headings with `df.rename()`.
* **Discretization (Binning)**: Converts continuous numbers into distinct categories using `pd.cut()`.
* **Date Extraction**: Breaks down datetime fields into isolated columns for Year, Month, Day, and Day of Week.
* **Feature Encoding**: Applies Label Encoding (ordinal lists) or One-Hot Encoding (binary column maps) to categorical text columns.
* **Feature Scaling**: Applies MinMaxScaler (scales values into `[0, 1]`) or StandardScaler (standardizes values to `mean=0, std=1`).

#### D. Analytics & Visualization (DASHBOARD Tab)
* **Dynamic Widget Canvas**: Mounts resizable, draggable widget boxes on a dashboard.
* **Recharts Visuals**: Generates bar charts, line plots, area charts, and scatter graphs from active datasets.
* **Metric Calculation**: Aggregates column details (sum, average, minimum, maximum, count) dynamically.

#### E. Machine Learning (ML STUDIO Tab)
* **Classification Models**: Trains K-Nearest Neighbors (KNN), Decision Tree, and Support Vector Machine (SVM) classifiers.
* **Regression Models**: Trains Linear Regression and Random Forest regressors.
* **Clustering Models**: Groups unstructured points into clusters using K-Means clustering.
* **Hyperparameters**: Tunes model behaviors via neighbor count (K), tree depth bounds, estimator count, and learning rates.
* **Evaluation Metrics**: Classifiers are evaluated using Accuracy, Precision, Recall, and F1-score; regressors use MSE and R² Score.
* **Feature Importances**: Extracts model coefficients and splits criteria to show feature contribution weights.
* **Inference Engine**: Allows real-time model evaluation using custom field inputs.

#### F. Deep Learning (DL STUDIO Tab)
* **ANN Model Builder**: Constructs a sequential Multi-Layer Perceptron (MLP) network by stacking node layers.
* **Layer Configurations**: Configures Dense layers, Dropout layers, and activation functions (ReLU, Sigmoid, Softmax).
* **Training & Compilation**: Compiles the model using optimizers (Adam, SGD), loss functions (MSE, Cross-Entropy), training epochs, and batch sizes.
* **Inference Console**: Feeds custom inputs into the compiled TensorFlow model to calculate real-time predictions.
